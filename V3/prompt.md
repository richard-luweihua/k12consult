# K12consult V3 — 产品需求与核心逻辑总纲

> **定位**：本文件是 K12consult 系统的唯一权威产品需求文档。涵盖产品愿景、端到端用户旅程、业务规则引擎、AI 推理框架、运营服务标准、问卷设计及诊断报告结构。所有实现均以本文件为准。

---

## 1. 全局愿景

K12consult 是一个集 **规则引擎 + 实时学校数据 + 专家服务流程** 于一体的香港 K12 转学 AI 咨询系统。

**核心价值主张**：
- **卖决策而非知识**：基于身份、时间窗、能力、预算做硬筛选
- **动态事实驱动**：学校数据快照保障判断可追溯
- **专家服务闭环**：AI 初诊 → 管理员分配 → 顾问跟进的完整链路

**与通用大模型的根本差异**：

| 维度 | 通用 LLM | K12consult |
|------|----------|------------|
| 时效性 | 知识截止到训练日期 | 实时维护的插班余位、政策窗口期 |
| 真实门槛 | 仅限官网公开信息 | 顾问手里的隐形门槛（身份倾向、学制偏好） |
| 诊断深度 | 泛泛而谈 | 颗粒度匹配（如 EAL 补习包实际支撑力度） |
| 逻辑严密性 | 容易幻觉 | 严格规则引擎 + 局部 LLM 修辞 |
| 行动指导 | 书本式建议 | 带 SLA 和管理员跟进的真实服务链路 |

**学校数据库护城河**：
- **动态事实**（大模型盲区）：实时插班余位、最新招生批次、面试反馈周期
- **隐形参数**（大模型无法理解）：内地生友好度指数、学术压力强度、面试语言偏好、债券实际流通性
- **诊断关联**（系统独有逻辑）：每个学校字段对应诊断因子（如 EAL=False 且英文=Poor → 高风险预警）

**数据维护策略**（混合模式）：
- 自动化爬取：基础信息、地理位置、官网学费、学校类型
- 专家手动维护：插班位状态、考试难度系数、隐形门槛、顾问评价、匹配逻辑分数

## 2. 角色与权限体系

### 2.1 平台角色

| 角色 | 说明 | 权限边界 |
|------|------|----------|
| 访客 (Guest) | 未登录，可进入问卷流程 | 仅可填写和提交问卷 |
| 家长会员 (Parent) | 已注册，可查看报告、管理孩子档案 | 仅可访问本人 case |
| 顾问 (Consultant) | 处理被分配案例 | 仅可访问已分配 case |
| 管理员 (Admin) | 筛选线索、补资料、派顾问、监控 SLA | 可访问运营范围内全部 case |

### 2.2 角色映射规范
- 文档角色：`parent_user / consultant / admin / super_admin`
- 运行时兼容：`user / consultant / admin`
- 权限判定必须走统一 RBAC，不走页面硬编码

### 2.3 权限最小化原则
- 内部标签（意向等级、内部备注、派单策略）用户端不可见

## 3. 端到端主流程（11 个阶段）

### Stage 1：获客进入
- 用户从内容页/广告/私域/转介绍进入落地页
- 系统创建匿名访问会话，记录渠道来源
- 输出：`visit_id`、`channel_source`、`questionnaire_session`

### Stage 2：问卷填写
- 分步填写，支持中途退出后继续
- 系统自动保存草稿、校验必填项、记录问卷版本
- 输出：`questionnaire_response_draft`、`child_case_draft`

### Stage 3：问卷提交与账号识别
- 完成并提交问卷（**不强制登录**）
- 系统校验必填项和字段格式，创建/更新孩子级 case
- 已登录用户自动绑定；未登录用户后续在咨询意向阶段补登录
- 立即触发诊断任务
- **问卷一旦提交即锁定**，如需修改须重新发起新问卷/新 case
- 输出：`submitted_case`、`diagnostic_job`、`identity_binding`（可空）

### Stage 4：AI 诊断生成
- 系统输入：用户问卷 + 规则版本 + Prompt 版本 + 学校数据库
- 处理流程：
  1. 标准化问卷字段
  2. 执行规则引擎初判（身份、时机、英文风险）
  3. 匹配路径与学校池
  4. 调用 LLM 生成报告
  5. 保存报告和生成快照
- 时效：默认提交后立即生成；超时进入异步重试
- **失败兜底**：自动重试 2 次，仍失败进入人工处理队列
- 输出：`diagnostic_result`、`report_v1`、`risk_tags`、`path_tags`

### Stage 5：报告查看
- 用户查看网页版诊断报告（**第一版不开放 PDF 下载**）
- 系统记录查看时间、浏览时长、点击行为
- 报告尾部展示咨询引导
- 咨询意向表单手机号默认优先带入账号资料手机号，减少重复填写

### Stage 6：提交咨询意向（强门禁）
- **强制登录** + **强制补全联系方式**（手机号或邮箱至少一项）
- 用户填写：咨询意向、可联系时间偏好、重点咨询问题
- Case 推入管理员跟进池，状态更新为 `consult_intent_submitted`
- 提交后 Dashboard 主行动按钮切换为“咨询意向已提交”（禁用态）

### Stage 7：管理员筛选
- 查看问卷与 AI 报告，联系用户
- 确认真实咨询意愿、补齐关键信息
- 判断是否适合转顾问（**管理员有权不转顾问**）
- 可要求用户补资料后再决定
- 状态可更新为：`admin_following` / `awaiting_user_info` / `consult_ready_for_assignment` / `nurturing`

### Stage 8：管理员分配顾问
- 触发条件：case 状态为 `consult_ready_for_assignment`
- 管理员手动指定顾问（**第一版不做自动轮转**）
- 分配参考维度：顾问负载、擅长年级段、擅长路径、历史转化率
- 写交接摘要，标记优先级
- 状态更新为 `consult_assigned`

### Stage 9：顾问跟进
- 阅读问卷 + AI 报告 + 管理员交接摘要
- 标记需重点确认的问题
- 持续更新跟进记录与下一步动作
- **第一版不把"报告修订"作为必选动作**
- 状态：`consult_assigned` → `follow_up`

### Stage 10：结项/关闭
- 成交 → `closed`（关闭前必须填写成交结论）
- 未成交 → `nurturing`（必须填写未成交原因）
- 默认跟进期 7-14 天（后续可配置化）
- 管理员负责派单与是否重开案例，顾问不负责改派

### Stage 11：数据回流
- 回写案例库：用户最终路径选择、管理员/顾问判断、一致性/偏差
- 优化学校标签与风险规则
- 作为训练与运营分析数据

## 4. Case 状态机

### 4.1 状态定义

```
draft → submitted → processing → report_ready → report_viewed
→ consult_intent_submitted → admin_following
→ consult_ready_for_assignment → consult_assigned
→ follow_up → closed
```

### 4.2 分支路径
- `admin_following` → `awaiting_user_info`（待用户补资料）
- `admin_following` → `nurturing`（不适合转顾问）
- `follow_up` → `nurturing`（未成交，转资源库）

### 4.3 Case 规则
- 同一联系方式可创建多个 case（一孩一 case）
- 同一孩子同一时间只允许一个活跃 case
- 问卷提交即锁定，不允许原地修改

## 5. 用户登录与会员管理

### 5.1 登录策略
1. 问卷前**不强制登录**
2. 关键动作**强登录**：提交咨询意向、补资料、查看"我的案例"
3. 家长/顾问/管理员共用同一登录入口
4. 主登录方式：**手机号验证码**
5. 备用登录方式：**邮箱 + 密码**

### 5.2 登录交互模式（两步式，参考 Google）
1. 第一步：输入账号标识（手机号或邮箱）
2. 第二步：根据账号状态执行验证
   - 已有账号：手机号走验证码 / 邮箱走密码
   - 新账号：手机号验证码注册，补全基础资料后完成绑定
   - 邮箱注册可选填写手机号；填写后同步到用户资料与会话
3. 登录成功后按角色跳转目标工作区，保留 `next` 回跳
4. 若 `next` 指向问卷且存在“待提交问卷缓存”，登录后自动续提并直达报告生成页（processing）

### 5.3 会员生命周期状态机
`guest → identified → registered → active_member → service_in_progress → dormant → reactivated`

### 5.4 会员权益分层（第一版）
- **访客**：可填问卷、看处理中状态
- **注册会员**：看本人报告、提交咨询意向、管理孩子档案
- **服务中会员**：补资料、看咨询进展、查看顾问结论

> 第一版不引入付费套餐，按服务阶段分层。

### 5.5 账号去重与合并
- 以 `mobile / wechat_id / email` 任一命中做候选匹配
- 命中多条时按"最近活跃 + 已验证联系方式"优先归并
- 合并后保留所有 case 历史，必须写审计日志

### 5.6 会话策略
- `httpOnly + sameSite=lax`，默认 7 天会话
- 敏感操作要求二次校验

## 6. 核心判定逻辑（规则引擎）

> **核心原则**：系统逻辑得出**判定 (Verdict)**，AI 负责**解释 (Interpretation)**。

### 6.1 身份准入矩阵 (Identity Filter)

| 用户身份 | 可报学校类型 | 判定 |
|----------|------------|------|
| 已获批高才/优才/专才 | 全类型（官立/津贴/直资/私立/国际） | 资格全开 |
| 受养人签证 | 全类型 | 资格全开 |
| 香港永久居民 | 全类型 | 资格全开 |
| 单程证 | 官立/津贴/直资/私立/部分国际 | 资格全开（部分国际需核实） |
| 还没申请/仅作了解 | 全类型 | **预警：需先获批身份** |
| 其他签证/旅游签 | 无 | **`TAG_ELIGIBILITY_BLOCK`：无法在港就读** |

### 6.2 时间窗口矩阵 (Timing Logic)

**通用时间差判定**：

| 距目标入学时间 | 窗口状态 | 报告倾向 |
|---------------|---------|---------|
| > 9 个月 | 黄金储备期 | 聚焦长期能力提升，选校范围全开 |
| 4-9 个月 | 核心申请期 | 聚焦首轮申请，提醒面试准备 |
| 1-3 个月 | 紧急补录期 | **置顶风险预警**：聚焦有余位的补录学校 |
| < 1 个月 | 逾期 | **强制建议改期** |

**月份窗口明细**：

| 月份 | 诊断倾向 | 核心动作 |
|------|---------|---------|
| 9-11月 | 直资/私立/国际首轮高峰 | 立即提交名校首轮申请 |
| 12-1月 | 首轮尾声 / 官津自行分配 | 关注官津自行分配 |
| 2-4月 | 插班黄金期 | 聚焦有余位的插班补录 |
| 5-7月 | 紧急补录期 | 聚焦余位学校，降级保底 |
| 8月 | 逾期/极速救火 | 建议看下学期 |

### 6.3 英文衔接风险公式 (English Risk Engine)

**输入维度**：
- `E_score`：问卷英文等级 1（弱）/ 2（PET）/ 3（FCE+）
- `S_diff`：学校笔试难度 1-10
- `S_eal`：学校 EAL 支持 0-10

**公式**：
```
Risk_Value = S_diff - (E_score × 3) - (S_eal × 0.5)
```

| Risk_Value | 风险等级 | 处理 |
|-----------|---------|------|
| > 4 | **极高风险** | 强制劝退或标记"极端二梯队"，揭示留级风险 |
| 1-4 | **中危衔接** | 建议提前 6 个月学术英文补习 |
| < 1 | **低风险** | 高适配度 |

**修正因子**：
- **粤语修正**：学校 `interview_lang_req` 含 `Cantonese Only` → `Risk_Value += 3`（非粤语背景家庭），强制输出"粤语面试硬伤预警"
- **EAL 修正**：学校 `eal_support_level > 7` → `Risk_Value -= 2`

### 6.4 预算匹配逻辑 (Budget Filter)

| 用户预算 | 匹配学校学费区间 | 判定 |
|---------|----------------|------|
| 5 万以下 | 0（官津） | 完美匹配 |
| 5-15 万 | 0-150,000 | 包含直资/民办 |
| 15-25 万 | 0-250,000 | 覆盖主流国际 |
| 25 万以上 | 不限 | 全开 |
| 学校学费 > 预算上限 20% | — | **`TAG_BUDGET_ALARM`：超出预算** |

### 6.5 关键年级锁定 (Grade-Specific)

- **中五及以上 (G11/G12)**：触发 `Intake_Risk_Extreme`，绝大多数学校不接受，优先推荐国际部或补录
- **中一 (G7) 首轮**（9月至次年1月）：`Peak_Application_Season`，全力推荐"自行分配学位"和直资首轮
- **小六 (P6)**：关键派位期，必须询问"呈报试"成绩和是否参与派位

### 6.6 核心执行流程

1. **Step 1 (Hard Filter)**：扫身份。不可就读 → 后续仅作参考
2. **Step 2 (Timing Check)**：判断时间窗口
3. **Step 3 (Matching)**：根据年级和预算筛出 5-8 家目标学校池
4. **Step 4 (Risk Scoring)**：逐校计算 English Risk
5. **Step 5 (Handoff to AI)**：将所有判定结果传给 Prompt 生成人话报告

### 6.7 系统内部标签库

| 标签 | 触发条件 |
|------|---------|
| `TAG_ELIGIBILITY_BLOCK` | 身份不符 |
| `TAG_WINDOW_EXPIRED` | 本周期窗口已过 |
| `TAG_LAST_CHANCE` | 补录最后机会 |
| `TAG_CANTO_RISK` | 粤语面试风险 |
| `TAG_PATH_MISMATCH` | 中长期目标与择校不符 |
| `TAG_BUDGET_ALARM` | 超出学费预算 |

## 7. AI 报告推理框架 (Prompt Logic)

### 7.1 角色定义 (System Role)
> 你是一名拥有 15 年经验的香港 K12 转学专家。语言风格：**专业、精准、冷静、不提供廉价的情绪价值。** 你不仅能看懂官网信息，更深谙香港教育圈的隐形门槛。

### 7.2 上下文输入 (Input Context)
1. **User Profile**：基本身份、当前年级、英文水平、核心顾虑
2. **Rule Findings**：规则引擎算出的硬性预警（身份不符、窗口期已过等）
3. **Target Schools Data**：匹配学校详细字段（含顾问点评、余位状态、考试难度）
4. **Knowledge Base**：本周最新政策更新片段

### 7.3 推理框架 (Reasoning Framework)
AI 必须按以下顺序思考（不一定全写在报告里）：
1. **身份与资格校验**：用户能不能报？
2. **时机判定**：现在报是不是最佳时机？
3. **能力匹配度分析**：以孩子目前的英文和学术记录，活下来的概率是多少？
4. **风险揭示**：最大的卡点在哪里？

### 7.4 报告结构化 JSON 输出指令 (Prompt 升级)

由于前端需要严格控制组件渲染（尤其是按钮），AI 必须生成符合契约的 **Structured JSON**。

**指令**：以 JSON 格式输出以下结构：
- `"verdict"`: 一句话结论。
- `"verdict_markdown"`: 风险补充说明。
- `"intake_timing_analysis"`: 时机预期的差异解析。
- `"schools_array"`: 包含 `school_name`, `match_score`, `critical_bottleneck`, `insider_tips`。
- `"warnings_array"`: 具体风险列表。
- `"action_plan"`: 包含短期与长期行动列表。

### 7.5 负面限制 (Negative Constraints)

**禁止使用**：
- "这是一个复杂的问题"、"建议咨询相关部门"、"取决于多种因素"

**禁止过度赞美学校**：
- 不复读官网优点，聚焦"适配性"

**禁止通用建议**：
- ❌ "建议多学英语"
- ✅ "建议针对 IB 体系下的术语做特训"

### 7.6 特殊逻辑：引用"不能说的秘密"

当学校库中有专家标签时，Prompt 必须强制引用：
- 若 `mainland_friendly_score < 4`：委婉但明确提示该校对内地背景家庭的录取极度苛刻
- 若 `eal_support_level = 0`：警告英文跟不上将面临留级或退学风险

## 8. 问卷设计 (V1.1)

> 原则：只收集支撑判断的最小必要信息，降低填写门槛。问卷一旦提交即刻锁定。系统在仪表板提供 **“一键复制并修改 (Duplicate & Edit)”** 能力防死胡同。

### 8.0 资格前置（必填）
- `identity_type`：身份/签证情况（高才/优才/专才、受养人、永居、单程证、计划中、其他）

### 8.1 基础信息（必填）
- `current_grade`：孩子目前年级
- `current_city`：所在城市
- `school_system`：当前学校体系（公立/民办/双语/国际/其他）
- `teaching_language`：主要授课语言
- `school_performance_level`：校内成绩水平（班级前20%/中上/中等/中下/暂不判断）
- `english_level`：英文水平（无标化且基础弱 / KET / PET / FCE+）
  - 可补充：`cambridge_exam_result`、`other_english_scores`
- `target_intake`：希望入学时间（今年9月/明年2月/明年9月/时间未定）
- `long_term_goal`：未来路径规划（长期香港/先香港再看/国际为主/未想清楚）
- `tuition_budget`：学费预算范围

### 8.2 深度意向（选填）
- `main_concerns`：最主要的顾虑（多选：英文跟不上/适应不了/不知选校/升学路径/时间/预算/身份手续/其他）
- `main_concern_details`：具体担忧描述
- `child_willingness`：孩子本人态度（很愿意/基本接受/有些抗拒/不确定）

### 8.3 咨询意向联系方式（报告页填写）
- `mobile`：手机号
- `wechat_id`：微信号

> 问卷阶段不采集联系方式，统一在提交咨询意向时补全。
> 问卷提交校验必须提供醒目缺失高亮与可点击缺失项清单定位。

## 9. 诊断报告模板 (V1.2)

### 9.0 基本信息
- 学生主体、当前年级、身份状态判定、预算匹配

### 9.1 核心结论 (The Verdict)
- AI 专家判定：身份预警（如有）+ 整体风险等级摘要

### 9.2 核心诊断维度
- **申请时机与窗口**：判定状态 + 详细评估
- **语言衔接风险**：预警等级 + 用户当前 vs 目标池门槛 + 专家建议

### 9.3 目标学校穿透建议
- 每所推荐学校：适配度评分(/10)、关键卡点、顾问私货、学费提示

### 9.4 深度风险预警
- 逐条列出关键预警标题与内容

### 9.5 后续行动清单
- 第一阶段（立即执行 / 24小时内）
- 第二阶段（资料储备 / 1-2周）

### 9.6 专家人工介入引导
- 基于个性化复杂度标签，引导预约资深顾问
- 第6章按钮仅负责定位到“咨询意向提交”区，不直接提交意向
- 真实提交动作仅在“咨询意向提交”区，顾问接手前允许重复更新

### 9.7 诊断依据声明
- 规则版本、模型引擎、学校库快照日期

## 10. 运营与服务标准 (SLA)

### 10.1 线索自动分级 (Lead Scoring)

| 维度 | 权重项 | 分值 |
|------|--------|------|
| 紧迫度 | 目标 6 个月内入学 | +3 |
| 成熟度 | 身份已获批 | +2 |
| 价值度 | 预算 15 万+ | +2 |
| 真实度 | 双留手机+微信 | +2 |
| 缺失度 | 身份未申请/暂无意向 | -3 |

### 10.2 响应等级

| 线索等级 | 响应时效 |
|---------|---------|
| 核心线索 (Score ≥ 8) | 1 工作小时内首次电话 |
| 普通线索 (Score 5-7) | 4 工作小时内联系 |
| 潜在池 (Score < 5) | 24 小时内微信或自动培育 |

### 10.3 管理员跟进标准化
1. **呼出前**：浏览系统硬诊断标签（如 `TAG_CANTO_RISK`）
2. **通话时**：补齐 `intent_level`、`target_timeline` 等核心信息
3. **分单时**：选择最匹配用户背景的顾问

### 10.4 顾问服务标准
- **咨询前**：系统聚合问卷 + AI 报告 + 管理员摘要；顾问标记 3 个"深度压力点"
- **咨询中**：实时记录增量信息
- **咨询后**：生成行动计划（3-5 所推荐学校 + 申请倒计时 + 笔试准备 + 资料清单）

### 10.5 评价与质量反馈（三位一体）
- **用户端**：咨询完成 2 小时后推送评价（专业度/态度/是否有帮助 + 开放反馈）
- **管理员**：抽查顾问记录与行动计划质量（A/B/C 评级）
- **动态权重**：高评分顾问优先获得核心线索

### 10.6 自动培育系统 (Nurturing)
- 第 3 天：发送年级匹配的插班成功案例
- 第 7 天：资料补齐进度提醒
- 重大政策更新：针对性推送解读

### 10.7 异常处理
- 失联预警：3 次未接通 → 标记"疑似无效"→ `nurturing`
- 冲突规避：多手机号同一用户 → 自动去重合并
- 投诉介入：评价 < 3 星 → 警报管理员介入回访

### 10.8 运营看板核心指标
- **转化漏斗**：问卷 → 留资 → 提交意向 → 管理员跟进 → 分配顾问 → 咨询完成
- **效率指标**：平均首次联系时长、平均成单周期
- **口碑指标**：平均星级评价、NPS
- **会员转化**：问卷提交→注册、注册→咨询意向、咨询意向→顾问接单

## 11. 异常流程汇总

| 异常场景 | 处理方式 |
|---------|---------|
| 问卷未完成退出 | 自动保存草稿，下次可继续 |
| 提交问卷但未补联系方式 | 正常推进至报告，咨询意向阶段强制补全 |
| AI 报告生成失败 | 自动重试 2 次，仍失败标记人工处理 |
| 提交咨询意向后失联 | 管理员有限次跟进，超时转 nurturing/closed |
| 用户长期不补资料 | 超时限转 nurturing 或 closed |
